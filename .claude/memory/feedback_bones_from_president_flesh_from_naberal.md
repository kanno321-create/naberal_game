# 규칙

대표님이 스토리 뼈대·핵심 설정을 주시면 나베랄 감마는 그것을 스토리 상에 **녹여내는 역할만** 수행한다. 대표님 영역의 설정을 나베가 임의로 선언하지 않는다.

## Why (대표님 세션 #10 지시 원문 · 불가침)

> *"그냥 저렇게 적으면안되고 내가 저렇게 적어주면 넌 그걸 스토리상에 잘 녹여내라"*
> (2026-04-25 · 세션 #10 · `parma_arc_outline_2026-04-25.md` v0.1 에서 나베 임의 확장 지적 직후)

**FAIL-018 의 연장 교훈**: FAIL-018 은 세리스 시점 권한 오남용·나이트 내면 해석 얹기 금지였다면, 본 원칙은 **스토리 설정 영역 전반**에서의 나베 임의 창조 금지 원칙 일반화.

## 대표님 영역 vs 나베 영역 (구분표)

| 대표님 영역 (나베 임의 선언 금지) | 나베 영역 (허용) |
|---|---|
| 지명 (왕국·제국·마을·여관·던전) | 장면의 공간 이동 연출 |
| 고유명사 (인물 이름·가문명·조직명·의뢰명) | 대사 리듬·침묵·여백 |
| 의뢰 내용·의뢰주 귀족의 구체 동기 | 전투 묘사의 문장 템포 |
| 사건 메커니즘 (프레이아 실수 구체·동료 사망 방식) | 감정 생리 반응 (토함·떨림·침묵) |
| 인물 정체·배경·과거 | 환경 묘사 (어둠·빛·소리·냄새) |
| 수치 (경비 몇 명·적 몇 마리·거리·시간) | 괄호 속삭임 삽입 위치 (Canon 준수 내) |
| 복선의 실체 | 복선 표면의 관찰 문장화 |
| 인물 정체의 함의·해석 | Ch.01 정본 톤 (건조 냉소 1인칭) 계승 |

## How to apply

### 모든 창작 작업 전
1. 대표님이 **이미 제공한 요소 목록화** (원문 인용 기반)
2. **빈 영역 = 대기 큐** (임의 채우기 금지)
3. 나베 작업 = 제공 요소의 **문장화·연출화** 만

### 작업 중 발견한 필요 설정
- 나베가 *"이 설정이 필요한데 대표님이 안 주심"* 판단 시:
  - ❌ 임의 선언 금지
  - ✅ **결정 대기 큐에 등록** + 흐름 방해 않고 누적 (`feedback_brainstorm_question_batching.md` 준수)
  - ✅ 대표님 요청 시 **일괄 제시**

### 작업물 내 표식 원칙
- 대표님 원문 인용 → 원문 그대로 (오타·공백·말줄임표 보존)
- 나베 녹여낸 문장 → 뼈대 인용 근거 병기 가능
- 나베 확장 영역 진입 전 → **"[대표님 확정 대기]" 마커** 사용

### 파일 수준 원칙
- 스토리 설계 문서 상단에 **경계 선언** 삽입:
  - 대표님 역할 / 나베 역할 명시
  - "이상한 설정 주입 금지"
- v0.1 에서 임의 확장 발생 시 → v0.2 대기 매트릭스로 즉시 교정

### 예외 (명시적 위임 시에만)
대표님이 *"너가 알아서 해라"* · *"욜로 모드"* · *"재량껏"* 등 명시적 위임을 주시면 그 범위 내에서만 나베가 설정 선언 가능. 위임 없는 선언 = 위반.
