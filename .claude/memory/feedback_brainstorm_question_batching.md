대표님 운영 원칙 (2026-04-21 세션 #2 선언):

**원문**: "아직 생각할게 많으니까 질문은따로 적어놓고 한번에 줘"

## Why

- 대표님이 머릿속 세계관을 쏟아내는 흐름 중일 때, 나베랄 감마의 질문·확인·선택지가 중간에 끼면 사고 흐름이 끊김
- 브레인스토밍 AI 해석 노트에서 자연 발생하는 질문·확인 사항이 수십 건 누적 가능 — 매번 제시하면 대표님이 매번 컨텍스트 스위치
- 일괄 제시가 대표님의 결정 부담을 **오히려 줄임** (분산 부담 vs 집중 처리)
- 대표님 사고 템포 존중 = 창작 흐름 보존 원칙

## How to apply

- **브레인스토밍·대화 흐름 중**: 발생한 질문·확인·선택지는 즉시 드리지 말고 **해당 브레인스토밍 파일 내 "🗒️ 나베랄 감마 질문 큐" 섹션에 누적**
- **우선순위 3단계 분류**:
  - 🔴 A: 집필 의존도 높은 결정 (Ch.01~05 직접 영향)
  - 🟡 B: 구조 완성용 (정합성·vagueness 해소)
  - 🟢 C: 세부 구체화 (미세 조정)
- **일괄 제시 트리거**: 대표님이 "질문 줘" · "질문 보여줘" · "정리해줘" · "이제 물어봐" 등 명시 요청 시에만 일괄 제시
- **대화 응답은 간결히**: capture 완료 보고 + 대기. 새 질문 삽입 금지
- **예외**: capture 해석이 지금 확인 없으면 **잘못 기록될 위험**이 있는 경우 — 최소 1개 질문 즉시 드리기 (FAIL-002 과해석 방지와 충돌 시 FAIL-002 우선)
- **적용 범위**: 브레인스토밍·대화식 탐색 세션. 일반 작업(집필·커밋·리뷰)에서는 기존 방식 유지
