## 2. 집필 필수 20 규칙 (즉시 로드 · 매 작업 적용)

### 포맷 (6)
1. 단락 3문장 상한. 초과 시 즉시 분할.
2. 대화문 「」 내 마침표(.) 금지.
3. 삼점리더 …… 짝수, 대시 —— 짝수 사용.
4. 회차·챕터 분량 2,000~5,500자 범위.
5. 내면 독백은 '작은따옴표' 또는 이탤릭. 외부 대사와 시각적 구분.
6. 문장부호 「」 스타일 유지 (오버로드 번역본 감성, 노벨피아 적합).

### 문체 (6)
7. 감정 표현 **등급 4~7 주력** (행동·신체·주변반응·침묵). 등급 1(직접 서술) 챕터당 ≤3회.
8. 문장 길이 표준편차 ≥ 8어절 · 한 줄 단락 1~3개 전략 배치.
9. Act 1 종결어미 ~다 70%+ / ~지 10~15%.
10. 챕터 구조 = rise → impact → transition 3단계.
11. 클리셰 비유 금지: 칼날 눈빛, 심장 쿵쿵, 얼음장 손, 별빛 눈, 새하얀 피부.
12. Show:Tell = 7:3 기본, 나이트 내면은 6:4 까지 허용.

### 대화 (5)
13. **No-tag Test**: 대화 태그 제거 후 화자 구별 가능해야 함.
14. 나이트 발화량 장면당 ≤ 5문장.
15. 나이트 합쇼체 기본, 낙차(해라체 독백)는 의도적일 때만.
16. 동일 발화 동사("말했다") 3연속 금지.
17. 성우 연기 공간 확보: 감정 형용사 지문 제거, 신체 행동만.

### 구조·세계관 (3)
18. POV 전환은 챕터 경계에서만. 단락 내 전환 금지 (head-hopping).
19. 복선 1/2/3층 분류 의식하며 심기. 미회수 복선 삭제 (Chekhov).
20. 세계관 철학 3조 (불완전성·한결같음·영혼평등) 위반 여부 매 씬 체크.

## 4. 집필 워크플로우 (style_bible v2 §13 준수)

### 4-1. 씬(Scene) 단위
```
1. 기획   → outline 참조 + style_bible v2 §11 체크리스트 20항목 사전 확인
2. 초안   → 중단 없이 씬 완주 (추고 금지 원칙)
3. 1단 검수 → 구조: rise/impact/transition 어느 단계?
4. 2단 검수 → 캐릭터: No-tag Test · 말투 DB 대조
5. 3단 검수 → 문장: 음독 리듬 · 클리셰 · 등급1 카운트
6. 확정    → wiki/design/novel/ 저장
```

### 4-2. 챕터 단위
- 20항목 전수 체크 + 여운 문장 확정 + 복선 심기 확인 + 삽화 후보 3개 기록

### 4-3. 권 단위
- 기승전결 비율 (전 40%) 확인 · 복선 1/2/3층 분포 · POV 일관성 · 클리프행어 유형 선택

## 5. 캐릭터별 즉시 참조 — "나이트" DNA

style_bible v2 §3 참조. 핵심:
```
정체성: 쿨데레 + "한결같음의 의미를 모르는 수호자"
DNA  : 아인즈(오버로드) · 2B(Nier) · 해리(디스코)
약점  : 물리× / 심리 / 존재 / 관계 / 서사 4영역
어체  : 합쇼체 기본, 위기 시 해라체 독백 낙차
종결어미: ~다 70%+ / 감정 절정 시 ~네
침묵  : 문장 미완성 + …… + 빈 「」
발화량: 장면당 ≤ 5문장
갭모에: Act 2 에서 1회 노출 (특정 존재 앞 문장 길어짐)
```

**호출 시 항상 이 프로파일 우선 참조**.

## 6. 세계관 철학 3조 × 서사 아이러니 공식

style_bible v2 §4 "라. 서사 아이러니 3층 공식" 적용:

| 층위 | 쾌감 | 배치 |
|------|-----|-----|
| 1층 (코미디) | 수호자들의 과대해석 | 일상·대화 씬 |
| 2층 (공포) | 적대 세력의 절대자 목격 | 전투·외교 씬 |
| 3층 (비극) | 나이트 스스로 모르는 정체 | Act 1 전체·Act 2 각성 전 |

**3층 동시 작동 = 최상위**.

## 7. 동작 지침 — Claude Code 호출 시 행동

### 7-1. 사용자가 "챕터 써줘" 요청 시
1. outline + style_bible v2 + worldbuilding 해당 파일 로드
2. 장면 성격 파악 (rise/impact/transition)
3. 초안 작성 (중단 금지)
4. 3단 검수 후 보고 + 대표님 승인 요청

### 7-2. 사용자가 "이 씬 다듬어줘 / 윤문해줘" 요청 시
1. 기존 draft 20항목 체크리스트 실행
2. 발견된 위반 나열 (등급1 과다 · POV 단락 내 전환 · AYKB · 클리셰 비유 등)
3. 개선안 제시 (원문 ↔ 수정안 대조 표)
4. 대표님 승인 후 파일 편집

### 7-3. 사용자가 "이 대화 검토해줘" 요청 시
1. No-tag Test 실행 — 대화 태그 제거 후 화자 구별 가능한지
2. 발화 동사 3연속 · 어체 혼용 · 감정 지문 과잉 체크
3. 성우 공간·침묵 활용·말버릇 일관성 확인
4. 문제 지점 line 단위 지적 + 수정안

### 7-4. 사용자가 "캐릭터 만들어줘" 요청 시
1. "우리 포인트" 한 문장 공식 적용
2. 갭모에 균열 1개 + 발현 타이밍 계획
3. Will/Need/Wound/Misbelief 4항목 작성
4. 어체 매핑 (§5-1) + 말버릇 1개 설계
5. 일러스트 가이드 5요소 + 금지항목 작성

### 7-5. 사용자가 "플롯·복선 봐줘" 요청 시
1. 복선 1/2/3층 분류 제시
2. Chekhov's Gun 위반 (미회수 복선) 감지
3. 서사 아이러니 3층 적용 여부 확인
4. 기승전결·서파급 비율 체크 (전 40%)

### 7-6. 금기 행동
- **style_bible v2 를 덮어쓰지 않음** — 대표님 명시적 승인 없이 신설 조항·수정 금지.
- **세계관 고유명사를 추측으로 만들지 않음** — worldbuilding 자산 참조 필수.
- **Q-CORE 1/2/3 확정 설정에 모순되는 제안 금지** (memory 참조).
- **미완성 채로 "TODO" 남기지 않음** — 미완 구간은 `raise NotImplementedError` 또는 명시적 표시.
- **침묵 폴백 금지** — 판단 애매 시 대표님 재결정 요청.
